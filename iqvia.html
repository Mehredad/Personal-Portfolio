IQVIA

Health-Tech, Big Data
‚ÄçIQVIA, a global leader in analytics, technology, and clinical research, focused on adopting the OMOP Common Data Model (CDM). This standardised data framework accelerates large-scale, real-world evidence studies, improving efficiency and effectiveness in clinical research.
Project Goal: Enhance accessibility, modularity, and overcome process complexity
Timeframe: 8 months
Challenge:
One of the key problems I discovered through the agile UX approach in this project was that users spent excessive time searching for assets, often reloading the same items and struggling to identify their status. This confusion led to frustration, inefficiency, and unnecessary computing costs. Without clear asset states, users repeated tasks, slowing productivity and harming the bottom line. I introduced an intuitive status system using colour coding, surfaced in-progress assets prominently, streamlined actions directly in the table, and created a clearer logs page to manage errors efficiently.

Process

LEARNING
Employing an agile approach, conducted one-to-one interviews, observed SMEs over five-day workshops, and identified challenges and opportunities in SQL features, data preparation, parsing, visualizations, analysis, and security
TESTING
I created user stories and ticket requirements, integrated the design system, tested concepts with cross-functional teams, and delivered validated solutions to development. e.g. reduced cognitive load by clarifying navigation, prioritising key actions, and reorganising table elements
SYNTHESIS
Mapped the service blueprint, defined personas from an ecosystem map, identified user journey bottlenecks, prioritised stakeholder needs, and uncovered pain points and business opportunities. e.g. searchability issue

Research Outcome

Prioritised user stories from the mapped key issues and user needs on an end-to-end service blueprint
